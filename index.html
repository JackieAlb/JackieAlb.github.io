基于Hexo+Github快速搭建个人博客

2017-10-18

小记：花了半天时间，基于Hexo和GitHub搭建了一个简单的静态个人博客，在网上参考了不少资料，但总觉不够系统和正确，下面实录自己的搭建过程，备忘和供同好参考

##一、工具准备 ##

    1. Hexo

Hexo是一个简单、快捷且功能强大的开源博客框架，支持Markdown语法，有大量的免费皮肤可以使用。

Hexo的安装十分简单，关键步骤如下，详细内容请参考Hexo官网：

$ npm install hexo-cli -g
$ hexo init blog
$ cd blog
$ npm install
$ hexo server
> 问题：Hexo Server启动后默认使用4000端口，在Windows上可能会使用无法访问问题，建议使用 -p port指定端口启动，本人使用的是8090端口，启动后效果如下图：
![Hexo 首次启动后页面](20171019-createPersonalBlog-md/hexo-server-index.png) - 2. GitHub个人博客及域名绑定 > GitHub为每个用户提供了1G的静态空间，可以做为个人博客空间，相关搭建方式，可以参考文章[利用GitHub建立你的个人网站](http://blog.csdn.net/u012422829/article/details/46285577)，这里不再赘述。 - 3. Markdown编辑器 > Markdown做为一种电子邮件风格的标记语言，具有纯文本的强兼容能力，可以使写作者专注文字而不是排版，格式转换方便，可以方便地转换为Html、PDF，并且有极好的标记语法可读性。随着GitHub、简书等内容平台的兴起，已经成为大部分博客写作的事实标准。Markdown的语法学习请走这个[传送门](http://www.jianshu.com/p/q81RER)。因为社区成熟，也有大量的编辑器工具可供使用，个人推荐 [MarkDownPad](http://www.markdownpad.com)（on Windows）和 [ByWord](https://bywordapp.com/)（on ios)。 ##二、主要步骤 ## - ### Hexo皮肤选择与安装 ### 打开Hexo网站，选择Themes,从中选择你喜欢的皮肤,进入GitHub上按作者给出的Readme进行安装，比如我选择的是[Jane](https://github.com/hejianxian/hexo-theme-jane)这个主题的皮肤，作者有详细的安装介绍，这里不做复述，请大家参考。 - ### 让你的Hexo与GitHub关联 ### 与GitHub关联是非常重要的一步，关系到后续你是否可以有效发布。 其实方法很简单，打开Hexo初始化的目录，按前文假定的Blog目录，打开根下的_config.yml文件，编辑下述内容 
deploy:
  type: git
  repo: https://github.com/yourGitHubAccount/yourGitHubAccount.github.io.git
> 注：Hexo其实可以关联很多不同的空间提供者，感兴趣的同仁可以自行学习探索。 - ### 撰写你的博文 ### 打开你的MarkdownPad或Byword开始你的第一篇博文吧！，记得把文章存储到Hexo初始化好的那个Blog目录下的./source/_posts/子目录下。 - ### 生成并发布 ### 用以下命令生成并上传发布： 
$ hexo clean
$ hexo generate
$ hexo deploy

注：我通常将这三个命令打包在一个shell批处理文件，并通过Schedule设定一个提交时间，如每天晚上9：00，这样每天就会自动将文章发布到个人博客站点的，自己仅关注把博文写得精彩充实即可。

写给自己

2017-10-17

一直是个很懒的人，起心动念时的诸多想法仅停留在大脑中，长时三五天，短时三五秒，便随着时光蹉跎而消失地无影无踪了。再一次了，又有了冲动，想将自己生活和学习中的点滴记录下来，一来自娱自乐，二来希望看到这些文字的同仁有所收获。

为了能够长久的坚持，下面摘录荀子《劝学》自勉。

积土成山，风雨兴焉；积水成渊，蛟龙生焉；积善成德，而神明自得，圣心备焉。故不积跬步，无以至千里；不积小流，无以成江海。骐骥一跃，不能十步；驽马十驾，功在不舍。锲而舍之，朽木不折；锲而不舍，金石可镂。蚓无爪牙之利，筋骨之强，上食埃土，下饮黄泉，用心一也。蟹六跪而二螯，非蛇鳝之穴无可寄托者，用心躁也。
